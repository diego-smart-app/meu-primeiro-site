Mas o que é um VCS?

Como próprio nome diz, versionamento de software é usado para controlar as versões do código fonte, ou do projeto como um todo. Por exemplo, você criou uma página web em HTML e CSS, o que faz inicialmene é criar uma pasta pra organizar seu projeto, se está funcionando sem problemas, então essa versão é salva, em outro momento alguma melhoria é feita, assim pra cada modificação é salvo uma nova versão (v0.1, v0.2, v1.3, etc). Desta maneira várias versões são criadas, aquela básica, a que melhorou, a que não funcionou por uma falha no programa entre outras. Nesse caso várias versões estão criadas, porém alguns problemas podem ocorrer, principalmente quando o projeto está sendo criado por uma equipe, pois são muitas pessoas atuando em partes diferentes no projeto. É necessário que cada um faça sua parte individualmente e caso ocorra modificações por pessoas diferentes, é de grande importância verificar e controlar a modificação correta, é aí que o versionador tem grande importância.

Quais vantagens de usar um VCS?

  • Controle de histórico
  • Trabalho em equipe
  • Ramificação do projeto
  • Segurança
  • Organização

O que são Git e GitHub?

Git e GitHub são coisas diferentes, mas com o mesmo propósito, de forma grosseira podemos dizer que ambos são repositórios versionadores, porém o GIT é local e o GitHub é remoto.

Breve história do GIT

O GIT foi criado em 2005 pelo criador do kernel do Linux, foi feito para atuar melhor que outros versionadores da época. A sigla definida por ele pode ser:

  • Três letras do alfabeto
  • Teimoso, cabeça-dura, pensa que sempre tá certo
  • Global Information Tracker
  • Goddamn idiotic truckload of shit

Breve história do GitHub

Criado em 2008 por um grupo de 4 pessoas, tem um código proprietário baseado no Git, porém foi comprado pela Microsoft em 2018 por míseros US$ 7,5 bilhões.

O GitHub e seu propósito

O GitHub é o repositório remoto, segue uma determinada sequência: usuário edita o código-fonte, faz um commit pro GIT (repositório local) e depois um push pro GitHub (repositório remoto).

O GitHub tem características de redes sociais, nesse repositório é possível armazenar, contribuir, ter uma referência pra oportunidades de trabalho, etc. Se resume também de uma grande comunidade para os programas open source.

Características do GitHub

  • Repositório ilimitados
  • Hospedagem de código-fonte
  • Características de rede social
  • GitHub Pages integrado (hospedagem simples de site)
  • Colaboração
  • Forks (continuar outro projeto respeitando a licença)

Criar uma conta no GitHub

Criar uma conta no é simples, basta acessar o link e fazer seu cadastro gratuito.

Vamos agora instalar as ferramentas necessárias para utilizar o repositório, são elas:

  • GitHub Desktop
  • GIT
  • VScode (poderia ser outro editor de código-fonte)

Usando as ferramentas

Passos da utilização das ferramentas

O primeiro passo é abrir o GitHub Desktop e criar um novo repositório (Create a New Repository on your hard drive...)

Depois de clicar, outra tela exibirá para que sejam preenchidas as informações do projeto e será explicado um a um.

  • Name

    Nome do seu repositório, evite colocar caracteres especiais e espaços.

  • Description

    Descrição da sua escolha, escreva uma breve apresentação do projeto, pra que será usado e principais características.

  • Local PATH

    Local que o projeto será armazenado no computador, vale ressaltar que por organização é importante deixar tudo numa mesma pasta. Ao fazer isso uma pasta com nome escolhido é criada nesse local. Clique no check box abaixo de local PATH, assim a descrição será exibida assim que o repositório for selecionado.

  • GIT ignore

    Deixe em branco, pois serve para o GIT ignorar se algum determinado tipo de arquivo for modificado.

  • Licence

    Essa parte é muito importante na questão jurídica, aqui é determinado o que outro usuário pode fazer com seu arquivo, você pode conferir os tipos de licenças aqui.

Depois de tudo preenchido é só clicar em criar. Essa ação criará uma pasta no diretório indicado em local PATH. Dessa é só criar ou colocar seu projeto novo na pasta citada. Toda vez que houver alguma alteração nos arquivos dessa pasta o GitHub Desktop irá indicar uma mudança e está pronto para fazer a atualização no repositório local e também o remoto, conforme imagem abaixo.

Na demonstração apresentada vemos uma mudança feita no repositório selecionado, nesse caso basta colocar um título da mudança e uma descrição, aí é só fazer um commit (atualização do repositório local). Depois um push para atualização do repositório remoto.
